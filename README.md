# Litopenaeus Vannamei Analysis

AI-powered web application for Pacific White Shrimp (*Litopenaeus vannamei*) detection, length measurement, weight estimation, and biomass analytics — built with vanilla HTML/CSS/JS on the frontend and Flask + OpenCV + scikit-learn on the backend.

## Tech Stack
- **Frontend:** HTML5, CSS3, vanilla JavaScript (no frameworks), Chart.js, jsPDF
- **Backend:** Python Flask (REST API)
- **Computer Vision:** OpenCV (grayscale conversion, noise removal, edge detection, contour detection, length measurement)
- **Machine Learning:** scikit-learn `RandomForestRegressor` (length → weight regression)
- **Database:** SQLite (`database.db`, created automatically on first run)

## Setup

```bash
# 1. Create a virtual environment (recommended)
python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# 2. Install dependencies
pip install -r requirements.txt

# 3. Train the weight-prediction model (one-time step, produces the .pkl file)
python train_model.py

# 4. Run the Flask app
python app.py
```

Then open **http://127.0.0.1:5000** in your browser.

> **Note on model training:** `train_model.py` is meant to be run locally or in a notebook/Colab environment purely to *produce* `models/shrimp_weight_model.pkl`. The Flask web app (`app.py`) never imports or calls Colab — it only loads the resulting `.pkl` file with `joblib` at startup. If you have real measured (length_mm, weight_g) data, drop it into `dataset/shrimp_measurements.csv` (same two columns) and re-run `train_model.py` to retrain on real data instead of synthetic data.

## How It Works

1. **Upload** — drag & drop one or more shrimp images (`POST /upload`). Files are stored in `static/uploads/` and registered in the SQLite `images` table.
2. **Analyze** (`POST /analyze`) — for each image:
   - OpenCV resizes the image, converts to grayscale, removes noise (Gaussian + median blur), runs adaptive thresholding + Canny edge detection, and extracts contours.
   - Each plausible shrimp-shaped contour is measured with `cv2.minAreaRect` to get its pixel length, which is converted to millimeters using the calibration factor: `length_mm = pixel_length × calibration_factor`.
   - The trained RandomForestRegressor predicts weight from length: `weight_g = model.predict(length_mm)`.
   - A confidence score is derived from contour shape heuristics (elongation + solidity).
   - Results (length, weight, confidence, bounding box) are saved to the `detections` table, and an aggregated row is saved to `analysis_results`.
3. **Dashboard** — the frontend renders four live stat cards (Total Detected, Average Length, Average Weight, Total Biomass), two Chart.js histograms (length & weight distribution), and per-image detection cards with bounding-box overlays.
4. **Export** — results can be exported as CSV or as a formatted PDF report (client-side, via jsPDF).

## API Endpoints

| Method | Route          | Description                                            |
|--------|----------------|----------------------------------------------------------|
| POST   | `/upload`      | Upload one or more images, returns stored image records |
| POST   | `/analyze`     | Run CV+ML pipeline on given `image_ids`, returns detections + stats |
| GET    | `/results`     | Fetch the most recent (or a specific `?batch_id=`) analysis batch |
| GET    | `/statistics`  | Aggregate statistics across all historical analyses      |
| GET    | `/health`      | Basic health/status check                                |

## Calibration

The pixel→millimeter conversion depends on your camera setup (distance, lens, resolution). Use the **calibration factor** field in the UI (mm per pixel) — calibrate it once by photographing an object of known length next to your shrimp samples and dividing its real length by its measured pixel length.

## Retraining with Real Data

Replace `dataset/shrimp_measurements.csv` with real measurements in the format:
```csv
length_mm,weight_g
85.2,3.1
102.7,5.4
...
```
Then run `python train_model.py` again — it will detect the existing CSV and train on it instead of generating synthetic data.
